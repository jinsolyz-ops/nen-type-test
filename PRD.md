# PRD.md — 서비스 요구사항 문서

## 1. 프로젝트 개요

**서비스명**: 念 Type Test (넨 타입 진단)  
**한 줄 설명**: 헌터헌터 세계관 기반의 넨 계통 진단 웹앱  
**목표**: 12개의 질문으로 6개 넨 계통 중 하나를 진단하고, 결과를 소셜 공유할 수 있는 바이럴 테스트 서비스

---

## 2. 사용자 시나리오

1. 랜딩 페이지에서 진단 시작
2. 12개 질문에 순서대로 답변 (4지선다)
3. 결과 페이지에서 내 넨 계통 확인
   - 계통 설명 카드 (능력·성격·직업 추천·대표 캐릭터 2명)
   - 6계통 오라 분포 레이더 차트
4. 트위터 또는 카카오로 결과 공유
5. 공유 URL로 접속한 사람도 동일한 결과 페이지 확인 가능

---

## 3. 핵심 기능 요구사항

### 3-1. 진단 테스트
- 총 12문항, 각 문항 4개 선택지
- 선택지마다 6개 계통에 가중치 점수 부여
- 선택 즉시 다음 문항으로 전환 (0.35초 딜레이)
- 진행 바로 현재 진행률 표시

### 3-2. 결과
- 가장 높은 점수의 계통이 결과
- 6계통 점수를 레이더 차트로 시각화 (애니메이션)
- 계통 상세 카드: 설명 / 주요 능력 / 성격 / 현실 직업 추천 3개 / 대표 캐릭터 2명(사진+능력 설명)

### 3-3. 공유
- 결과마다 고유 URL 생성 (`/result/[shareId]`)
- 트위터(X) 공유: Web Intent + OG 이미지
- 카카오 공유: Kakao SDK 피드 공유 + OG 이미지
- 링크 미리보기용 OG 이미지 자동 생성 (1200×630)

### 3-4. 트래킹
- 페이지별 방문 수 기록
- 테스트 완료(플레이) 수 기록

## 5. 비기능 요구사항

- 모바일 우선 반응형
- 초기 로딩 3초 이내
- Vercel Edge로 OG 이미지 즉시 생성
- 공유 URL은 영구 유효 (Supabase 데이터 보존)

---

## 6. 범위 외 (Out of Scope)

- 로그인/회원가입
- 결과 히스토리
- 관리자 대시보드
- 다국어 지원
